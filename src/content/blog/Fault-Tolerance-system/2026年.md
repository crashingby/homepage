## 解释一下容错系统中Fault,Error,Failure

#### **① Fault（故障 / 隐患 / 根源）**

- **定义：** 系统的**底层缺陷或物理异常**。它是静态存在的，是导致问题发生的**根本原因（Root Cause）**。
    
- **特点：** Fault 存在于系统内部，在没有被触发时，它是隐蔽的，系统此时看起来依然在正常运行。
    
- **分类：**
	* _硬件 Fault_：比如内存条的一个电容老化、硬盘出现坏道、微处理器受到宇宙射线干扰（Bit Flip）。
    
    - _软件 Fault_：也就是我们常说的 **Bug**（例如代码里写了除以 0 的逻辑，或者存在空指针隐患）。
        

#### **② Error（错误 / 内部状态异常）**

- **定义：** 由于 Fault 被触发，导致系统内部**状态偏离了正确值**。它是 Fault 的**具体表现形式**。
    
- **特点：** Error 是系统内部的异常。如果系统具备“容错能力”（比如有重试机制、纠错码 ECC、或者备份部件），Error 就可以在内部被消化或修复，**不一定会暴露给最终用户**。
    
- **例子：** 内存中某个原本应该是 `0` 的比特位，因为射线干扰变成了 `1`。此时系统内部的数据已经“脏了”（发生了 Error），但还没打印到屏幕上。
    

#### **③ Failure（失效 / 系统崩溃 / 结果）**

- **定义：** 系统内部的 Error 没能被及时拦截或修复，最终蔓延到了系统边界，导致系统**无法再提供预期的服务**。
    
- **特点：** Failure 是**用户可见的**灾难。它标志着整个系统或某个服务的瘫痪。
    
- **例子：** 网站页面显示 `500 Internal Server Error`，或者ATM机直接死机吐不出钱。
