# 博客写作约定

这个目录记录当前博客系统支持的 Markdown 写法，避免以后忘记项目里的特殊约定。

## 文章放在哪里

博客文章统一放在：

```txt
src/content/blog/<分类 slug>/<文章 slug>.md
```

例如：

```txt
src/content/blog/Fault-Tolerance-system/2025年.md
src/content/blog/leetcode/dp学习.md
```

其中：

- **分类 slug** 是文件夹名，也是 URL 中的分类名。
- **文章 slug** 是 Markdown 文件名去掉 `.md` 后的名字，也是 URL 中的文章名。
- 分类的中文显示名在 `src/lib/blog.ts` 的 `topicConfig` 中配置。

## Front Matter

每篇文章开头可以写元信息：

```md
---
title: 2025年真题
date: 2026-06-14
tags: [容错系统, 真题]
summary: 容错系统 2025 年真题整理与解题笔记。
---
```

当前支持的字段：

- `title`：文章标题，会显示在博客列表和文章页。
- `date`：文章日期，用于博客列表排序。
- `tags`：文章标签，会显示在文章卡片和文章页顶部。
- `summary`：文章摘要，会显示在博客列表和文章页标题下方。

## 标题导航

文章页右侧目录会读取正文里的标题：

```md
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

当前目录显示 `h1` 到 `h4`。正文里的 `h1` 到 `h6` 都会自动生成跳转锚点。

## Obsidian 风格链接

博客支持类似 Obsidian 的 wiki link 写法。

链接到另一篇文章：

```md
[[2022年]]
```

链接到另一篇文章的某个标题：

```md
[[2022年#什么是马尔科夫过程？马尔科夫过程满足泊松分布的前提是什么？]]
```

自定义显示文字：

```md
[[2022年#什么是马尔科夫过程？马尔科夫过程满足泊松分布的前提是什么？|参考 2022 年马尔科夫过程]]
```

解析规则：

- 优先在当前文章所在分类中查找目标文章。
- 如果当前分类里没找到，再去 整个 blog 下所有文章里找
- 目标文章可以写文件名 slug，也可以写 front matter 中的 `title`。
- `#` 后面的标题文字必须和目标文章中的标题文字一致。
- 代码块中的 `[[...]]` 不会被解析，所以 C++ 的 `[[nodiscard]]` 不会被误伤。

## 常规 Markdown 能力

当前博客支持：

- GitHub 风格表格和任务列表。
- LaTeX 行内公式：`$O(N)$`。
- LaTeX 块级公式：

```md
$$
R(t) = e^{-\int_0^t z(x) dx}
$$
```

- Mermaid 图：

````md
```mermaid
graph TD
    A --> B
```
````

- 代码高亮，推荐写明确语言：

````md
```cpp
int main() {
    return 0;
}
```
````
